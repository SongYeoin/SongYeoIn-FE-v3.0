# type: Subject (아래에 작성/ 최대 50글자/ 서술형X, 간결&요점 서술/마침표 및 특수기호 X)

# Body(아래에 작성/ 한 줄 당 72자 내로 작성/ 무엇을 변경했는지 또는 왜 변경했는지를 설명)

# Footer(아래에 작성)

##################
# ✨feat: 새로운 기능 추가
# 🐞fix: 버그 수정
# ✏️chore: 빌드 업무 수정, 패키지 매니저 수정, production code와 무관한 부분들 (.gitignore, build.gradle 같은)
# 🎨design: CSS 등 사용자 UI 디자인 변경
# 🔨refactor: 코드 리팩토링
# ✅test: 테스트 코드, 리팩토링 테스트 코드 추가
# 📃docs: 문서 수정
##################
# 영어로 Subject 작성 시
# Add : 추가
# Remove : 삭제
# Simplify : 단순화
# Update : 보완
# Implement : 구현
# Prevent : 방지
# Move : 이동
# Rename : 이름 변경
##################
# Fixes: 이슈 수정중일 때 사용 (아직 해결되지 않은 경우)
# Resolves: 이슈를 해결했을 때 사용
# Ref: 참고할 이슈가 있을 때 사용
# Related to: 해당 커밋에 관련된 이슈번호 (아직 해결되지 않은 경우)
##################
# 예시
# feat: "로그인 기능 구현"

# 로그인 시 JWT 발급

# Resolves: #111
# Ref: #122
# related to: #30, #50
